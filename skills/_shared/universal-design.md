# ユニバーサルデザイン（UD）リファレンス

`ux-ui-audit-expert` (検出・gate・post-check 担当) と `designer-expert` (Design Plan 設計・実装担当) の
両方が参照する、ユニバーサルデザインの原則と検出基準。

- ux-ui-audit-expert は本ドキュメントを **検出基準** として用いる (broken_invariant の根拠付け)
- designer-expert は本ドキュメントを **設計指針** として用いる (Design Plan の Accessibility Requirements)

---

## UD 7原則（Ron Mace, 1997）

| # | 原則 | 概要 | UI/UXへの適用 |
|---|------|------|-------------|
| 1 | **公平な利用** | 多様な能力のユーザーに同等の体験を提供 | スクリーンリーダー対応、キーボード操作、色覚多様性対応 |
| 2 | **利用の柔軟性** | 多様な好みと能力に対応 | テーマ切替、フォントサイズ変更、入力方法の多様性 |
| 3 | **単純で直感的** | 経験・知識・言語に関わらず理解可能 | 明確なラベル、アイコン+テキスト、一貫したパターン |
| 4 | **認知できる情報** | 感覚能力に関わらず必要な情報を伝達 | 多感覚フィードバック（視覚+聴覚+触覚）、色以外の情報伝達手段 |
| 5 | **うっかりミスへの寛大さ** | 意図しない操作の悪影響を最小化 | Undo機能、確認ダイアログ、自動保存 |
| 6 | **少ない身体的努力** | 最小限の疲労で効率的に利用 | 大きなタッチターゲット、ショートカット、入力補助 |
| 7 | **適切なサイズと空間** | 体格・姿勢に関わらず操作可能 | 48dp以上のタッチターゲット、十分な要素間隔 |

## WCAG 2.2 主要基準（Web/Appの UD 実装指針）

### レベル A（最低限）

| 基準 | 内容 | 検出方法 |
|------|------|---------|
| 1.1.1 | 非テキストコンテンツに代替テキスト | `<img>` に `alt`、アイコンに `aria-label` |
| 1.3.1 | 情報と関係性をプログラムで判定可能 | セマンティックHTML、`role` 属性 |
| 1.4.1 | 色だけで情報を伝えない | エラー表示にアイコン+テキスト併用 |
| 2.1.1 | キーボードで全操作可能 | `@click` 要素に `@keydown` 対応 |
| 2.4.1 | ブロックスキップの手段 | スキップリンク、ランドマーク |
| 2.4.2 | ページタイトルが適切 | `<title>` / `useHead()` |
| 3.1.1 | ページの言語が指定 | `<html lang="ja">` |
| 4.1.2 | 全UIコンポーネントに名前と役割 | `aria-label`、`role`、セマンティック要素 |

### レベル AA（推奨）

| 基準 | 内容 | 検出方法 |
|------|------|---------|
| 1.4.3 | コントラスト比 4.5:1（通常テキスト） | テキスト色と背景色のペア検証 |
| 1.4.4 | テキストサイズ200%拡大に対応 | `rem`/`em` 使用、固定px回避 |
| 1.4.11 | 非テキストUIのコントラスト比 3:1 | ボーダー、アイコン、フォーム要素の色 |
| 2.4.7 | フォーカスインジケーター可視 | `focus-visible` スタイル定義 |
| 2.5.8 | ターゲットサイズ 24×24px 以上 | ボタン、リンクの最小サイズ |
| 3.2.3 | ナビゲーションの一貫性 | 全ページで同じ位置にナビ |
| 3.3.1 | エラー箇所の特定 | フィールドレベルのエラー表示 |
| 3.3.2 | ラベルまたは説明文 | 全入力にラベル/プレースホルダ以外の説明 |

### レベル AAA（理想）

| 基準 | 内容 | 検出方法 |
|------|------|---------|
| 1.4.6 | コントラスト比 7:1 | 高コントラストモード |
| 1.4.8 | テキストの視覚的提示（行間1.5倍以上、段落間隔1.5倍以上） | `line-height`、`margin-bottom` |
| 2.2.3 | タイムアウトなし（セッション除く） | 自動ログアウトの警告 |
| 2.3.1 | 3回/秒以上の点滅なし | アニメーション制御 |

## 色覚多様性対応

### 色覚タイプと影響

| タイプ | 人口比率 | 影響 | 対策 |
|--------|---------|------|------|
| **1型（P型）** Protanopia | 男性約1% | 赤が暗く見える、赤-緑の区別困難 | 赤単独で状態を示さない |
| **2型（D型）** Deuteranopia | 男性約5% | 赤-緑の区別困難（最多） | 緑=成功、赤=エラーに必ずアイコン/テキスト併用 |
| **3型（T型）** Tritanopia | 約0.01% | 青-黄の区別困難 | 青と黄の隣接配置を避ける |
| **全色盲** | 極めてまれ | 明度のみで認識 | 明度差を十分に確保 |

### 色覚安全な配色ガイドライン

1. **赤と緑を対で使わない** — 成功/エラーを色だけで区別しない
2. **色+形+テキストの3重符号化** — 情報を色だけに依存させない
3. **明度差を確保** — グレースケールでも区別可能であること
4. **パターンや模様の併用** — グラフでは色だけでなくパターンも使用
5. **カラーシミュレーション検証** — P型/D型シミュレーションで確認

### セマンティックカラーの UD 対応

```
✕ 色のみ:     🔴 → エラー    🟢 → 成功
◯ 色+アイコン: ⚠️ エラー発生   ✅ 保存完了
◎ 色+アイコン+テキスト: ⚠️ エラー: メールアドレスが無効です
```

## キーボード・支援技術対応

### キーボードナビゲーション必須要件

| 項目 | 要件 | 検出パターン |
|------|------|------------|
| Tab順序 | 論理的な順序でフォーカス移動 | `tabindex` の適切な使用（`tabindex > 0` は避ける） |
| フォーカストラップ | モーダル内でフォーカスが循環 | モーダルの `focusTrap` 実装 |
| スキップリンク | メインコンテンツへのスキップ | ページ先頭の `<a href="#main">` |
| Escape | モーダル/ドロップダウンを閉じる | `@keydown.esc` ハンドラ |
| Enter/Space | ボタン操作 | `<button>` 使用（`<div @click>` ではなく） |

### ARIAライブリージョン

動的コンテンツの変化をスクリーンリーダーに通知する:

| 属性 | 用途 | 例 |
|------|------|-----|
| `aria-live="polite"` | 非緊急の更新通知 | トースト通知、リスト更新 |
| `aria-live="assertive"` | 即座に通知すべき変更 | エラーメッセージ、アラート |
| `role="status"` | ステータス更新 | 保存完了、ローディング完了 |
| `role="alert"` | 重要な警告 | バリデーションエラー |

## モーション・アニメーションの UD

### prefers-reduced-motion 対応

```css
/* 基本: アニメーションを定義 */
.element { transition: transform 200ms ease-out; }

/* UD対応: reduced-motion 時はアニメーション無効化 */
@media (prefers-reduced-motion: reduce) {
  .element { transition: none; }
}
```

### Tailwind での対応
```html
<div class="transition-transform duration-200 motion-reduce:transition-none">
```

### Flutter での対応
```dart
// MediaQuery で確認
final reduceMotion = MediaQuery.of(context).disableAnimations;
```

### 点滅・フラッシュの制限

- **3回/秒以上の点滅を禁止**（てんかん発作のリスク）
- `animation-iteration-count: infinite` のアニメーションは `prefers-reduced-motion` で停止すること

## テキスト・タイポグラフィの UD

| 項目 | 基準 | 理由 |
|------|------|------|
| 最小フォントサイズ | 16px（本文） | 視力低下者の可読性確保 |
| 行間 | 1.5倍以上 | 読字障害（ディスレクシア）への配慮 |
| 文字間隔 | 調整可能 | `letter-spacing` で拡張可能であること |
| 段落間隔 | 行間の1.5倍以上 | 読みやすさの確保 |
| テキスト揃え | 左揃え推奨 | 均等割付（justify）は単語間隔が不均一になる |
| フォント選択 | サンセリフ推奨 | 装飾フォントは読字障害に不利 |

## タッチ・ポインタの UD

| 項目 | 基準 | 根拠 |
|------|------|------|
| **最小タッチターゲット** | 48×48dp（推奨）/ 44×44px（WCAG） | 運動機能障害、高齢者への配慮 |
| **ターゲット間隔** | 最低8dp | 誤タップ防止 |
| **ドラッグ操作の代替** | タップ/ボタンで同等の操作が可能 | 細かい運動が困難なユーザー |
| **ジェスチャーの代替** | スワイプ→ボタン、ピンチ→+/-ボタン | 複雑なジェスチャーが困難なユーザー |
| **長押しの代替** | コンテキストメニュー / 明示的ボタン | 長押し操作が困難なユーザー |

## UD重大度判定基準

| 重大度 | 基準 | 例 |
|--------|------|-----|
| **致命的** | 特定ユーザーが全く操作できない | キーボード操作不可、スクリーンリーダーで認識不可 |
| **高** | 操作可能だが著しく困難、またはデータ損失リスク | コントラスト不足、タッチターゲット過小、色のみの情報伝達 |
| **中** | 利用可能だが体験が明確に劣る | ARIA属性不足、フォーカス順序不適切、代替テキスト不足 |
| **低** | 改善推奨だが現状でも大きな問題にはならない | AAA未達成項目、装飾的改善 |

## UD チェックの自動化支援

### 静的解析ツール（参考）

- **axe-core** — WCAG 自動テスト（Vue/React対応）
- **eslint-plugin-vuejs-accessibility** — Vue a11y リント
- **flutter_accessibility** — Flutter a11y チェック
- **Lighthouse Accessibility** — Chrome DevTools の a11y 監査

### コードレベルでの Grep 検出パターン

| 検出対象 | Grep パターン | 対象 glob |
|---------|-------------|----------|
| alt 欠如の img | `<img(?![^>]*alt)` | `*.vue`, `*.tsx`, `*.html` |
| div/span のクリック | `<div.*@click\|<span.*@click\|<div.*onClick` | `*.vue`, `*.tsx` |
| outline 削除 | `outline-none\|outline: none\|outline: 0` | `*.vue`, `*.css`, `*.scss` |
| tabindex 不正 | `tabindex="[2-9]\|tabindex="[1-9][0-9]` | `*.vue`, `*.tsx`, `*.html` |
| aria-label 欠如のボタン | `<button(?![^>]*aria-label)(?![^>]*>[^<])` | `*.vue`, `*.tsx` |
| lang 属性欠如 | `<html(?![^>]*lang)` | `*.html` |
| 色のみの情報伝達 | `color:.*red\|color:.*green`（文脈で判断） | `*.vue`, `*.css` |
| reduced-motion 未対応 | `animation\|transition`（`prefers-reduced-motion` の有無と対比） | `*.css`, `*.scss` |
